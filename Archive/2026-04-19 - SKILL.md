# Math Book MyST Migration

This skill manages the end-to-end workflow for converting chapters from the LaTeX math book
"All Math All The Time" into MyST Markdown articles and publishing them to GitHub Pages.

## Project Facts

| Item | Value |
|------|-------|
| LaTeX source | `~/Dropbox/Apps/Overleaf/All Math All The Time/` |
| Main LaTeX file | `A_All Math All The Time.tex` (use for Part/Chapter order) |
| MyST config | `myst.yml` in the Dropbox folder (single source of truth) |
| GitHub repo | `https://github.com/gcodekhanna1/all-math-all-the-time` |
| Live site | `https://gcodekhanna1.github.io/all-math-all-the-time` |
| Local repo clone | `~/Dropbox/all-math-all-the-time-git/` (permanent clone тАФ do not re-clone) |
| Publish script | `~/.claude/skills/math-book-myst-migration/scripts/publish.sh` |

**Key principle:** Dropbox is the single source of truth. All `.md` files live alongside their `.tex` counterparts in the Dropbox folder. GitHub is purely a publishing mechanism.

## Authoring Environment

Gaurav writes and edits LaTeX in **Overleaf**. Overleaf syncs to Dropbox in real time тАФ any
change saved in Overleaf is immediately available in the local Dropbox folder.

**VS Code does not auto-detect these external file changes.** After editing in Overleaf, refresh
VS Code manually: right-click the Explorer panel тЖТ **Refresh Explorer**.

The sandbox reads directly from the filesystem and always sees the latest Dropbox version тАФ no refresh needed.

## Workflow Overview

```
1. READ     тЖТ Read the .tex chapter file from Dropbox (after Overleaf sync)
2. CONVERT  тЖТ Produce the .md file using the conversion rules below
3. SAVE     тЖТ Write .md to Dropbox (same folder as .tex)
4. UPDATE   тЖТ Uncomment the file in myst.yml TOC + update index.md
5. PUBLISH  тЖТ Run publish.sh to copy files to GitHub repo clone and push
```

## Content Fidelity

The `.md` file is a faithful rendering of the `.tex` тАФ not a paraphrase, not a summary.
Every word, every sentence, and every equation must be reproduced exactly as written in the `.tex`.
Only the *markup* changes.

**The `.tex` is always the content authority.** If the `.md` and `.tex` ever diverge in prose
or equations, the `.tex` wins. Any prose improvement should be made in the `.tex` first (in
Overleaf), then reflected in the `.md` тАФ never the other way around.

**What changes in conversion (markup only):**
- LaTeX formatting commands are stripped or converted to MyST equivalents
- Math environments are rewrapped (`\begin{equation}` тЖТ `$$`)
- Custom environments become MyST admonitions
- Figures become MyST figure directives or placeholders
- Constructs with `% MyST: skip` are intentionally dropped

**What never changes:**
- The actual prose тАФ every word, every sentence, in the exact order written
- Equations тАФ reproduced symbol-for-symbol, just in MyST math syntax
- The structure тАФ sections, subsections, and their order

## Global Skip and Conversion Rules

These rules apply universally across all chapters. **No annotation is needed in the `.tex`** тАФ just apply them automatically during conversion.

### Always Strip (remove entirely, no replacement)

| LaTeX construct | Reason |
|----------------|--------|
| `\setlength{\jot}{...}` | Math spacing control тАФ irrelevant on web |
| `\nonumber` | Equation numbering suppression тАФ not needed in `$$` blocks |
| `\index{...}` | Index entries тАФ no index in MyST version |
| `\vphantom{...}` | Invisible vertical spacer тАФ layout only |
| `\hphantom{...}` | Invisible horizontal spacer тАФ layout only |
| `\boldmath` / `\unboldmath` | Not supported in MathJax; render math normally |
| `\tcbset{...}` | Raw tcolorbox style settings тАФ no MyST equivalent |
| `\hfill\tikz{...}` (decorative end markers in examples) | Decorative triangle/symbol тАФ skip entirely |

### Always Convert (same way every time, no annotation needed)

| LaTeX construct | MyST equivalent |
|----------------|-----------------|
| `\vab{x}` | `\|x\|` (absolute value) тАФ standardized globally |
| `\begin{empheq}[box=...]...\end{empheq}` | Plain `$$` block тАФ strip the box styling |
| `\makebox[\textwidth][r]{$\S$}` | `<div style="text-align:right">┬з</div>` |
| `\url{https://...}` | `[https://...](https://...)` |
| `\textrm{...}` inside math | Keep as-is тАФ MathJax supports it |
| `\dfrac` | Keep as-is тАФ MathJax supports it |
| `\displaystyle` inside math | Keep as-is тАФ MathJax supports it |

### Author Notes тАФ Always Skip

Any text that is clearly a note to the author rather than content for the reader should be **skipped entirely** тАФ do not include it in the `.md`.

Gaurav standardizes author notes with a `TO DO` label. Patterns to skip:
- Any `\textcolor{...}{...}` block whose content is an editorial note or reminder
- Any text containing `[TO DO`, `[EXPAND`, `[Come up with`, `[NOT SURE`, `[A brief history`, or similar bracketed editorial markers
- Stub sections that contain only a note to the author with no actual content

## LaTeX Annotation Convention

During conversion, insert MyST annotations directly into the `.tex` file wherever a LaTeX
construct requires a non-obvious conversion decision. This makes future re-conversions
mechanical rather than requiring fresh judgment each time.

**Only annotate where needed.** Standard constructs (inline math, equations, section headings,
basic text formatting) do not need annotations тАФ they convert cleanly by rule. Annotate
package-specific environments, custom macros, figures, and anything that has no direct
MyST equivalent.

**Annotation format** тАФ always placed immediately above the LaTeX construct it applies to:

```latex
%
% MyST:
% <instruction>
% <comment explaining why>
%
\the-latex-construct-below
```

**Instruction values:**

- A MyST directive to use тАФ e.g., `:::{admonition} Example\n:class: example`
- `skip` тАФ drop this construct entirely in the .md
- `placeholder` тАФ insert a TODO placeholder (used for figures without a rendered PDF yet)
- A plain English note if the situation is more complex

**Examples:**

```latex
%
% MyST:
% skip
% \pagebreak has no web equivalent
%
\pagebreak
```

```latex
%
% MyST:
% :::{admonition} Example
% :class: example
% tcolorbox 'example_elegant_1' тЖТ MyST admonition
%
\begin{example_elegant_1}
```

```latex
%
% MyST:
% placeholder тАФ see circle_tangent_angle_1.pdf when rendered
% TikZ inline; no standalone PDF yet
%
\begin{tikzpicture}
```

Annotations are added during the first conversion and updated in place if the conversion
strategy changes. Each annotation can differ even for the same package тАФ maximum flexibility
is intentional.

## Step 1 & 2: LaTeX тЖТ MyST Conversion Rules

When converting a chapter, first scan the `.tex` file for `% MyST:` annotations and follow
their instructions. Then apply the general rules below for everything else.

### Document Structure

| LaTeX | MyST |
|-------|------|
| `%!TEX root = ...` comment | Remove entirely |
| `%%%%...%%%%` comment dividers | Remove entirely |
| `\chapter{Title}` + `\label{chap:label}` | Frontmatter: `title: Title` + `label: chap:label` |
| `\section{Title}` + `\label{sec:label}` | `(sec:label)=\n## Title` |
| `\subsection{Title}` + `\label{sub:label}` | `(sub:label)=\n### Title` |
| `\setcounter{examples}{0}` | Remove |
| `\newpage` | Remove |
| `\noindent` | Remove |
| `\vspace{...}` | Remove |

**Frontmatter template for every converted file:**
```yaml
---
title: "NN - Chapter Title"
label: chap:label_from_latex
author: Gaurav Khanna, Ph.D.
numbering:
  enumerator: "N.%s"
---
```

Replace `NN` with the zero-padded chapter number (e.g. `01`, `02`, `10`) and `N` with the
plain number for the enumerator (e.g. `1`, `2`, `10`). The `title` field drives the left-hand
nav label; the `enumerator` drives equation/figure/section numbering within the chapter.

Example for chapter 2:
```yaml
title: "02 - Numbers - A Gentle Introduction"
numbering:
  enumerator: "2.%s"
```

**Chapter Number Registry** тАФ update this table each time a new chapter is converted:

| # | File | Title |
|---|------|-------|
| 1 | `foundations_what_is_the_meaning_of_this` | 01 - Introduction |
| 2 | `foundations_numbers_a_gentle_introduction` | 02 - Numbers - A Gentle Introduction |

If a chapter is ever split or renumbered, update only the `enumerator` value in the
affected files' frontmatter тАФ prose and cross-reference labels never need to change.

### Text Formatting

| LaTeX | MyST |
|-------|------|
| `{\bf text}` or `\textbf{text}` | `**text**` |
| `{\it text}` or `\textit{text}` | `*text*` |
| `\href{url}{text}` | `[text](url)` |
| `\textcolor{color}{text}` | `text` (drop color) |
| `---` (em dash in text) | `тАФ` (Unicode em dash) |
| `\epigraph{quote}{attribution}` | Blockquote (see below) |

**Epigraph format:**
```markdown
> "Quote text here"
>
> тАФ *Attribution text*
```

### Math

| LaTeX | MyST |
|-------|------|
| `$inline math$` | `$inline math$` (unchanged) |
| `\begin{equation}\n...\n\end{equation}` | `$$\n...\n$$` |
| `\begin{equation}\label{eq:label}\n...\n\end{equation}` | `$$\n...\n$$ (eq:label)` |
| `\begin{aligned}...\end{aligned}` inside equation | Keep as-is inside `$$` block |
| `\eqref{eq:label}` | `{eq}`eq:label`` |

### Cross-References

MyST **automatically prepends** the type label (Table, Figure, Equation, Section) when rendering a `{ref}`. Therefore, **never include the type word before a `{ref}`** тАФ doing so causes "Table Table 1", "Figure Figure 2", etc.

| LaTeX | MyST |
|-------|------|
| `Figure \ref{fig:name}` | `{ref}`fig:name`` |
| `Table \ref{tab:name}` | `{ref}`tab:name`` |
| `Eqn.(\ref{eq:name})` or `Eqn.(\ref{eq:name})` | `{ref}`eq:name`` |
| `Section \ref{sec:name}` | `{ref}`sec:name`` |
| `Chapter \ref{chap:name}` | `{ref}`chap:name`` |
| bare `\ref{fig:name}` | `{ref}`fig:name`` |

### Lists

| LaTeX | MyST |
|-------|------|
| `\begin{itemize}...\end{itemize}` | `- item` (unordered list) |
| `\begin{enumerate}...\end{enumerate}` | `1. item` (ordered list) |
| `\begin{enumerate}[label=(\alph*)]` | `a. item` (alpha list) |
| `\item text` | `- text` or `1. text` |

### Custom Theorem Environments (tcolorbox)

These are the custom environments from `elegant.cls`. Map each to a MyST admonition:

| LaTeX environment | MyST admonition |
|------------------|-----------------|
| `\begin{definition_elegant_1}...\end{definition_elegant_1}` | `:::{admonition} Definition\n:class: tip\n...\n:::` |
| `\begin{theorem_elegant_1}[Name]...\end{theorem_elegant_1}` | `:::{admonition} Theorem (Name)\n:class: important\n...\n:::` |
| `\begin{example_elegant_1}...\end{example_elegant_1}` | `:::{admonition} Example\n:class: example\n...\n:::` |
| `\begin{exercise_elegant_1}...\end{exercise_elegant_1}` | `:::{admonition} Exercise\n:class: exercise\n...\n:::` |
| `\begin{yellowbox_noborder_bluesidebar_1}...\end{yellowbox_noborder_bluesidebar_1}` | `:::{note}\n...\n:::` |
| `\begin{tcolorbox}[title=Title]...\end{tcolorbox}` | `:::{admonition} Title\n...\n:::` |

**Important:** The content inside these environments may itself contain math, lists, or other LaTeX тАФ apply all conversion rules recursively.

### Figures

**Strategy:** All figures are being migrated to standalone TikZ `.tex` files that output PDFs. Not all are there yet. Handle as follows.

**Canonical asset location:** All figure files live in the `figures/` subfolder of the Dropbox folder. This is a strict rule with no exceptions тАФ there is no fallback to the root folder. All `.md` figure references must include the `figures/` prefix. Decorative images (chapter headers, portraits) go in `document_images/` and are referenced as `document_images/filename.jpg`.

**Label-to-filename convention:** Almost all figures and TikZ environments have a `\label{fig:name}`. The label name and the corresponding file in `figures/` are very similar or identical. Use the label to determine the filename тАФ e.g., `\label{fig:heirarchy_of_numbers_1}` тЖТ look for `figures/heirarchy_of_numbers_1.pdf` (or `.png`, `.jpg`, `.svg`). There is no ambiguity.

**Decision rule:**
- File exists in `figures/` subfolder тЖТ reference it in the `.md` with `figures/` prefix
- File does not exist тЖТ standalone figure not yet created; insert a placeholder and note which file needs to be moved to `figures/`

**Pre-conversion step:** Before converting a chapter, move all its figure assets from the Dropbox root into `figures/`. This must be done before running the publish script.

1. **`\includegraphics{filename}` or `\includesvg{filename}`** тАФ check if `figures/filename.png` (or `.jpg`, `.svg`) exists. Always reference the `.png` version in `.md` files тАФ **never `.pdf`** (browsers cannot render PDFs inline). The `publish.sh` script auto-converts PDFs to PNGs, so the PNG will exist after the first publish.
   ```markdown
   :::{figure} figures/filename.png
   :alt: Description
   Caption text here.
   :::
   ```

2. **`\begin{tikzpicture}...\end{tikzpicture}` inside a `\begin{figure}` or `\begin{tabular}`** тАФ read the `\label{}` to determine the expected filename. Check if that file exists in `figures/`. If yes, reference the `.png` version. If no, insert a placeholder:
   ```markdown
   :::{figure} figures/TODO_figurename.png
   :alt: TODO
   **[Figure placeholder тАФ standalone PDF not yet created]**
   :::
   ```

3. **`\begin{figure}[...]` environments** тАФ extract the `\includegraphics` or TikZ inside and apply rules above. Use the `\caption{}` content as the figure caption.

### Tables

**Tables without captions/labels** тАФ convert `\begin{tabular}` to a plain markdown table:
```markdown
| A | B | C |
|---|---|---|
| 1 | 2 | 3 |
```

**Tables with captions and/or labels** тАФ use the `:::{table}` directive. The caption goes in the directive title, the label as `:label:`:
```markdown
:::{table} Caption text here
:label: tab:label_name

| A | B | C |
|---|---|---|
| 1 | 2 | 3 |

:::
```

If a table has a label but no caption, use an empty title:
```markdown
:::{table}
:label: tab:label_name

| A | B | C |
|---|---|---|

:::
```

Strip all LaTeX table formatting: `@{}`, `m{...}`, `c`, `l`, `r` column specs, `\toprule`, `\midrule`, `\bottomrule`, `\hline`, `\hspace{...}`. Keep math content in cells as-is.

Run the publish script using the Bash tool:

**To publish all modified chapters at once:**

2. Copy `myst.yml`, `index.md`, and all `.md` files from Dropbox to the local clone

**Asset organization (canonical тАФ no exceptions):**

**Pre-publish step:** Before running the script, make sure all figure assets for the chapter being published are in the `figures/` subfolder in Dropbox тАФ not in the root.

GitHub Actions then builds and deploys automatically. The live site updates in ~1 minute at:

## Things to Watch For

- **Duplicate labels:** LaTeX warns about these; MyST does not. If the source compiled cleanly in LaTeX, labels should be unique. Still worth a quick scan.
- **`\textcolor{...}{...}`:** Drop the color wrapper but keep the text. Colors don't translate to web without custom CSS.
- **Multi-part environments:** Some tcolorbox environments span multiple pages in LaTeX (using `breakable`). In MyST, admonitions don't page-break тАФ this is fine.
- **Nested environments:** Math inside theorem boxes, lists inside examples тАФ apply conversion rules recursively.
- **`\cite{...}` references:** Bibliography not yet implemented in MyST version. Convert to a footnote or inline attribution for now, and flag it with a `<!-- TODO: citation -->` comment.
